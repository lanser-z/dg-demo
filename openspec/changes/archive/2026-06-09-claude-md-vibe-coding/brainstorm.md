# brainstorm — claude-md-vibe-coding

## Feedback

| 角色 | 核心反馈 |
|------|----------|
| 开发者（本人） | CLAUDE.md 应**先读文档**，**最小知识进 CLAUDE.md**，**其他知识点用文件索引**——按需读取，不一次性灌进上下文 |
| 开发者（本人） | 当前 CLAUDE.md 已偏离该原则：把 `docs/` 里的内容大量复制过来（项目结构、技术栈、命令、设计决策、注意事项），导致 context window 浪费、单点修改漂移（docs 改了 CLAUDE.md 没跟上） |
| 开发者（本人） | docs 是项目前期工作和经验总结的**唯一权威源**（6 个文档，约 110KB），CLAUDE.md 不应与之并列存在 |

**核心原则（来自反馈）**：
1. docs 是知识权威源，每次会话**先**读相关文档
2. CLAUDE.md 只放"不带文件就读不到"的最小上下文
3. 任何"可指向文件"的内容，**必须**用文件索引代替内联

---

## Ideas

- [ ] **Idea 1**：把 CLAUDE.md 改为"最小内核 + 文件索引"结构
  - 内核（≤30 行）：项目是什么、必读顺序指针、与 docs 的关系约定
  - 索引：按"任务场景"组织的文件指针表（生成数据→看哪个文件、调试质量问题→看哪个文件、跑演示→看哪个文件）

- [ ] **Idea 2**：删除 CLAUDE.md 中所有与 docs 重复的内容
  - 现有 121 行中约 60 行属于"复制粘贴 docs"（技术栈、命令、设计决策、注意事项）
  - 这些都应改为文件索引或不写

- [ ] **Idea 3**：把"必读顺序"从硬编码流程改为"按任务路由"
  - 现有 5 步必读顺序对所有场景都强加阅读成本
  - 改为：先看 Background.md（业务背景是通用前置），再按当前任务选读其他文件

- [ ] **Idea 4**：在 CLAUDE.md 顶部明示"读文档约定"
  - 告诉 Claude Code：**不要假设 CLAUDE.md 之外的内容已知**，按需 `Read` 文档
  - 防止 Claude 跳过文档直接猜

- [ ] **Idea 5**：保留"全局硬约束"作为最小内核的一部分
  - 例如：`data/` 不入 git、演示环境内存 ≥16GB、`uv` 是唯一包管理器
  - 这些约束如果不显式声明，Claude 容易违反

## Value

| Idea | 影响的非功能属性 | 价值描述 | 是否值得转为变更？ |
|------|----------------|----------|-------------------|
| 1    | 可维护性、context 效率 | CLAUDE.md 从 ~120 行降到 ~40 行，每次会话省 80 行 context；新增/修改知识只改 docs 不改 CLAUDE.md | ✅ 是 |
| 2    | 可维护性、单点真理 | 消除双份维护负担，避免 doc 与 CLAUDE.md 漂移 | ✅ 是 |
| 3    | 效率、用户体验 | 不同任务读不同文件，避免"改一个 bug 必须先读 5 篇 doc" | ✅ 是 |
| 4    | 正确性 | Claude 主动按需读文档，避免基于过时/缺失上下文瞎猜 | ✅ 是 |
| 5    | 正确性 | 硬约束显式声明（如不要 `pip install`、不要 commit data/），减少错误操作 | ✅ 是 |

**结论**：5 个 Idea 都是同一个变更的组成部分，合并为 1 个 change 即可。
