## Ideas

- [ ] **Idea 1：把第 7 节（re-upload 演示）从教学 notebook 剥离**——7.1~7.7 是 OpenSearch 脚本与 GMS GraphQL 的开发者操作混入了资产可视化课。整体移到独立的 `notebook/datahub_setup.ipynb`（面向 dev/运维），主 notebook 7.1 处只留一行引用。
- [ ] **Idea 2：在第 1 节前加一个"痛点故事"cell**——用 1 分钟的剧本演示"新员工小王要找个煤质化验数据"的两幕对照：没有可视化时（问 3 个同事、下载错表、被领导骂）vs 有可视化后（搜「煤质」1 分钟定位、看到 Owner 与质量分）。回答「为什么要做」。
- [ ] **Idea 3：每条质量告警旁加一句 `[业务影响]` 白话翻译**——把"0.5% 重复"翻译成"5 万条/年 × 单条对账 5 分钟 = 208 工时 ≈ 2.5 万成本"；"PI 瓦斯 0.28% 危险告警"翻译成"瓦斯超 1% 就有爆炸风险，按 0.28% 漏报率算每年漏 X 起"。回答「跟我有什么关系」。
- [ ] **Idea 4：加一节"DataHub 是什么、怎么用、与 notebook 的关系"**——介绍 DataHub 是什么（公司级元数据中心）、用 UI 能做什么（搜资产、看 Owner、看 Schema、看血缘）、与本 notebook 的边界（本 notebook 是离线分析，DataHub 是线上协作平台）。回答「这套东西我以后怎么用」。
- [ ] **Idea 5（备选）：在资产目录后加一个"数据在哪、长啥样"cell**——给每张表打印前 3 行示例 + 字段类型，让小白**看到数据**而不只是看到「18,105,000 行」这种抽象数字。
- [ ] **Idea 6（备选）：每个 cell 末尾加一句"🎯 教学小结"**——把每个 section 浓缩成 1 句话，帮小白边学边记。结尾加 1 张"3 句话带走"总结卡。
- [ ] **Idea 7（备选）：把第 2 节"5 个异构系统"扩展为"为什么是 5 个、有没有可能 6 个"**——给小白一个心智模型"治理不是套模板，是要先盘清家底"。

## Plan

### 立即实现（Idea 1+2+3+4，本期完成）

- **Idea 编号**：1, 2, 3, 4
- **初步方案**：
  1. 抽出 step1.ipynb 第 7.1~7.7 → 新建 `notebook/datahub_setup.ipynb`，在原位置留 1 行引用
  2. 在 step1.ipynb 第 1 节前插入 1 个 markdown cell 写「痛点故事」剧本
  3. 在第 5 节（详细质量告警）每条告警后追加 `[业务影响]` 一行；需要先调研 5 个系统的真实量级（年订单量、年化验批次、年传感器告警量等）
  4. 在第 6 节（模块总结）前插入 1 节「DataHub 是什么、怎么用」，含：UI 截图、3 个最常用操作（搜索资产 / 看 Owner / 看 Schema）、与 notebook 边界说明
- **预计总耗时**：4 小时（Idea 3 调研占大头）
- **关键依赖**：
  - DataHub UI 当前 7 容器 healthy，UI 截图可直接 `playwright screenshot` 获取
  - 5 个系统真实量级在 `data/historical/` 下的 Parquet 文件可查（每张表行数 × 注入率）
  - 业务影响换算参考 `docs/Background.md` 中已记录的业务场景（煤化工单价、安全事故代价）
